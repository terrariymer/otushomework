Исходя из моей схемы столбцы с высокой кардинальностью поставил бы на первое место в таблицах.

Можно добавить сосатвной  индекс для ускорения запроса:
CREATE INDEX idx_product_price_name
ON products(price, name);

Можно установить ограничения NOT NULL во все таблицы по рекомендации Дмитрия Кириллова,а после при небходимости убирать где действительно нужен будет NULL.
Так же можно добавить ограничения UNIQUE в столбцы : "Phone number, Website, Vendor code", чтобы у нас не было повторяющихся значений.
